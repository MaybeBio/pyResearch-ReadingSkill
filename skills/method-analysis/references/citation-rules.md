# 双标签系统与引用格式

## 双标签系统

| 标签 | 含义 | 使用场景 |
|-----|------|---------|
| `[原文声明]` | 论文Method章节明确描述的方法 | 原文中的方法步骤/架构/配置 |
| `[模型归纳]` | 基于原文的分析推理 | 归纳方法特点/优劣势 |
| `[模型归纳·方案推演]` | 基于原文外推的方案组件 | 缝合方案中新组合的方法模块（不是原文中已有的） |
| `[开源实现]` | 从仓库README提取的实现信息 | 仓库名/API/环境依赖 |
| `[未明确给出]` | 原文中未提及的方法细节 | 参数/配置/实现细节缺失 |
| `[不确定]` | 信息有歧义 | 方法描述不清晰 |

## 不确定性标记

| 标记 | 含义 |
|------|------|
| `[未明确给出]` | 论文中未提及的字段 |
| `[不确定]` | 信息有歧义 |
| `[无开源实现]` | 未找到对应的开源仓库 |
| `[仓库未维护]` | 仓库存在但已停止更新 |

## 方法引用格式

### Method原文引用
```
[原文声明] 原文描述「The model uses a 3-layer MLP with 256 hidden units and ReLU activation」（PMID 12345678, Method）
[模型归纳] 该MLP架构简单但有效——3层是轻量级选择，256隐藏单元适合中等规模输入特征。
```

### 方案推演引用
```
[模型归纳·方案推演] 缝合方案：将Disobind的outer product思想应用于IDBSpred的MLP分类器前，可同时编码IDR-partner的序列互补性。
推理依据：
  1. Disobind（PMID 39763873）展示了outer product捕捉序列间互补性的有效性
  2. IDBSpred（PMID 41959048）的MLP可接收拼接后的向量输入
  3. → outer product输出 + ESM-2 embedding拼接 → MLP
```

### 仓库README引用
```
[开源实现] 「The model checkpoint is available at huggingface.co/xxx」
  仓库: owner/repo-name
  环境依赖: Python 3.8+, PyTorch 1.12+, transformers 4.x
  对应论文: PMID 12345678
```

---

## 引用三要素

所有方法引用必须包含：
1. **来源文献**：DOI/PMID作为唯一标识
2. **原文内容**：保留英文，用「...」包裹
3. **来源标注**：Method 或 README

---

## 原文-方案推演配对原则

**`[原文声明]` 是方法证据，`[模型归纳·方案推演]` 是缝合创新。二者缺一不可。**

- 每个方案组件必须有原文出处
- 缝合方案（新组合）标注 `[模型归纳·方案推演]` + 推理依据
- 禁止虚构原文中不存在的架构/参数/结果

## 仓库引用格式

使用 `<<<PY_PAPERFLOW_REPO_BOUNDARY>>>` 作为仓库分隔符。

```
[开源实现] 「README中的方法描述」
  仓库: owner/repo-name
  关键API: [主要函数/类名称]
  对应论文: PMID 12345678
```
