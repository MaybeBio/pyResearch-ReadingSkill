# 方法缝合规则与策略

## 6种缝合策略

所有策略均基于 `method_analysis.md` 中的idea产生方法论，每策略含操作模板和示例。

---

### 策略1：组合 (Combination) ⭐ 主要策略

**定义**：将两个或多个已有方法的独立模块拼接为新的完整方案。

**操作模板**：
```
[组合方案] 新方案名称
├── 模块A：来自 [方法1] (PMID XXXXXXXX)
│   └── 原文描述：「原文中的方法模块A描述」
│   └── [模型归纳] 该模块在新方案中的作用和适配性
├── 模块B：来自 [方法2] (PMID YYYYYYYY)
│   └── 原文描述：「原文中的方法模块B描述」
│   └── [模型归纳] 该模块在新方案中的作用和适配性
└── 缝合点：模块A的输出→模块B的输入，需适配：[具体适配方案]
```

**适用场景**：
- 两个方法的模块功能互补（如一个擅长特征提取，另一个擅长分类）
- 两个方法处理不同但相邻的子问题

**示例**：
```
[组合方案] IDR-PPI双向预测器
├── 模块A：IDR侧 → 来自Disobind的partner-dependent embedding (PMID 39763873)
├── 模块B：Partner侧 → 来自IDBSpred的ESM-2 residue-level classifier (PMID 41959048)
└── 缝合点：Disobind的outer product输出→IDBSpred的MLP输入，通过交叉注意力桥接
```

### 策略2：迁移 (Migration)

**定义**：将一个领域成熟的解决方案迁移到另一个领域。

**操作模板**：
```
[迁移方案] 问题：[目标问题]
原领域：[来源领域] 方法：[源方法] (PMID XXXXXXXX)
  └── 原文描述：「原文中的方法描述」
目标领域：[目标领域]
  └── [模型归纳] 两个领域的相似性分析 + 需要适配的差异点
适配策略：[具体的迁移适配方案]
```

**适用场景**：
- 两个领域有共享的抽象结构（如蛋白质序列 vs 自然语言文本）
- 来源领域的方法比目标领域更成熟

**示例**：
```
[迁移方案] LLM的RLHF对齐方法→IDR结合位点预测的偏好优化
原领域：NLP中的RLHF/DPO对齐
目标领域：IDR结合位点预测
  └── [模型归纳] IDR结合偏好可类比为语言模型的生成偏好——正确的结合位点是"被偏好的输出"
适配策略：将IDR-partner互作对作为"偏好对"训练目标
```

### 策略3：逆向 (Reversal)

**定义**：反问已有结论的反面或默认假设是否成立。

**操作模板**：
```
[逆向方案] 原假设：[原文中的默认假设] (PMID XXXXXXXX)
逆向问题：「如果这个假设不成立/反过来，会怎样？」
  └── [模型归纳] 逆向后的新视角 + 可能揭示的新现象
新方案：[基于逆向视角的方案设计]
```

**适用场景**：
- 已有领域共识但缺乏验证
- 默认假设限制了方法设计空间

**示例**：
```
[逆向方案] "更多数据→更好模型" 的逆向
逆向问题：在IDR-PPI数据极度稀缺的情况下，是否少而精的标注数据比重而糙更有效？
  └── [模型归纳] IDR互作数据的标注难度极高，质量vs数量的trade-off需要重新评估
新方案：主动学习策略→优先标注信息量最大的IDR-partner对
```

---

### 策略4：细化 (Refinement)

**定义**：通过增加限制条件收缩研究焦点，找到精准切入点。

**操作模板**：
```
[细化方案] 大问题：[原始问题]
万能句式：「在[大问题]中，针对[特定条件]，研究[具体方面]」
细化约束：[条件1], [条件2], [条件3]
  └── [模型归纳] 每个约束如何缩小问题空间，增大可行性和创新密度
细化后的问题：[精准问题陈述]
```

**适用场景**：
- 大问题已被广泛研究，需要细分赛道
- 新手入门、快速出活

**示例**：
```
大问题：如何预测IDR介导的蛋白互作？
细化语句：「在IDR介导的PPI预测中，针对多亚基复合物场景，研究协同折叠机制的计算建模方法」
约束1：多亚基（不是binary）→ 排除Disobind等binary方法
约束2：协同折叠→ 需要动力学信息，排除纯静态方法
约束3：计算建模→ 不需要实验方法
```

### 策略5：扩展 (Extension)

**定义**：在已有方法的基础上改进特定不足。

**操作模板**：
```
[扩展方案] 基础方法：[已有方法] (PMID XXXXXXXX)
原文中作者承认的局限性：
  └── [原文声明] 「局限性描述」(PMID XXXXXXXX)
  └── [模型归纳] 该局限性的影响范围和严重程度
改进方向：[针对该局限性提出的具体改进]
与基础方法的差异：[量化/定性的改进目标]
```

**适用场景**：
- 已有方法的局限性明显且被作者承认
- 改进有明确的评估指标

**示例**：
```
[扩展方案] Disobind的PTM扩展
基础方法：Disobind (PMID 39763873)
原文局限性：「our method cannot assess PTM effects as the pLM does not distinguish modified amino acids」
改进方向：替换pLM为PTM-aware的变体（通过微调引入磷酸化位点token）
差异：新增PTM感知能力，预测磷酸化如何改变结合界面
```

---

### 策略6：评估创新 (Evaluation Innovation)

**定义**：自创评估指标或多角度评估体系，避开卷SOTA。

**操作模板**：
```
[评估创新方案] 评估维度：[不止于精度]
新增评估指标：
  ├── [指标1]：[定义+计算公式+合理性论证]
  ├── [指标2]：[定义+计算公式+合理性论证]
  └── [指标N]：[定义+计算公式+合理性论证]
  └── [模型归纳] 为什么这些新指标能揭示已有指标忽略的信息
评估Protocol：[多角度评估流程]
```

**适用场景**：
- SOTA竞争激烈难以超越
- 已有评估指标忽略了重要维度
- 新问题尚无标准评估指标

**示例**：
```
[评估创新方案] IDR-PPI预测的多维度评估
新增指标：
  ├── 结合构象合理性得分：预测的结合位点在已知复合物结构中是否合理
  ├── PTM敏感性：方法对磷酸化位点附近的预测是否与实验一致
  └── 折叠能力预测准确率：预测的"foldable"区域是否确实可折叠
[模型归纳] 已有指标（MCC/AUC）仅评估结合位点预测精度，未评估构象合理性和功能相关性
```

## 缝合策略选择指南

| 场景 | 推荐策略 | 原因 |
|------|---------|------|
| 有2+互补方法模块 | 组合 | 直接拼接，风险最低 |
| 跨领域类比 | 迁移 | 借鉴成熟方案 |
| 挑战默认假设 | 逆向 | 可能产生突破性创新 |
| 大问题已被广泛研究 | 细化 | 快速找到精准切入点 |
| 已有方法有明确不足 | 扩展 | 局部改进，可控性强 |
| SOTA难以超越 | 评估创新 | 改变游戏规则 |

## 缝合质量检查

- [ ] 每个缝合模块有原文来源(PMID/DOI)
- [ ] 每个缝合模块有原文引用
- [ ] 缝合点有具体的适配方案
- [ ] 缝合策略明确标注
- [ ] 无虚构的方法组件
- [ ] 可实现性评估合理
