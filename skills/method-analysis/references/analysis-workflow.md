# 分析流程详细说明

## 三阶段分析流程

---

## 第一阶段：问题定义与拆解

### 1. 列出dc-analysis全部开放问题

从dc-analysis输出中提取开放问题清单，按严重程度排序展示：

```
📋 从dc-analysis继承的Y个开放问题：
OP-1: [问题描述] — 严重程度：高/中/低
OP-2: [问题描述] — 严重程度：高/中/低
...
```

### 2. 用户指定问题

用户用自然语言描述目标问题。支持4种模式：
- **选择模式**："我要解决 OP-1"
- **杂糅模式**："我要把 OP-1 和 OP-3 结合起来解决"
- **自定义模式**：用户完全自定义的问题描述
- **混合模式**："基于 OP-1，但要额外考虑PTM的影响"

### 3. 问题精确化

将用户问题拆解为子问题树：

```
核心问题：[用户的核心问题陈述]
├── 子问题1：[子问题描述]
│   ├── 输入：[什么数据/特征]
│   ├── 输出：[什么结果/预测]
│   └── 约束：[什么限制条件]
├── 子问题2：[子问题描述]
│   ├── 输入：[...]
│   ├── 输出：[...]
│   └── 约束：[...]
└── 子问题N：[...]
```

每个子问题标注：可操作性评估、是否已有部分方案、需要哪类方法支持

### 4. 主题对齐

标注问题涉及的intro-analysis主题：

```
问题主题映射：
├── 核心主题：[主题X] (from intro-analysis)
├── 相关主题：[主题Y], [主题Z]
└── 跨主题交叉地带：[描述]
```

---

## 第二阶段：全域方法检索

### 1. Method章节检索

从Method提取md中检索与问题相关的方法组件。**不限主题标签，全域文献都可用。**

检索维度：
- **功能维度**：数据处理方法、特征提取方法、模型架构、训练策略
- **问题维度**：与子问题直接相关的方法
- **主题维度**：核心主题 + 相邻主题的方法

### 2. 仓库README检索

从仓库README汇总md中检索开源实现：
- 搜索 `<<<PY_PAPERFLOW_REPO_BOUNDARY>>>` 分隔的每个仓库
- 提取：仓库名、功能描述、API接口、依赖环境、使用示例
- 标注每个仓库对应的论文PMID/DOI

### 3. 按Pipeline阶段归类

```
数据层：
  ├── [方法A] (PMID XXXXXXXX) — 开源: [仓库名]
  ├── [方法B] (PMID YYYYYYYY) — 未开源
  └── ...

特征层：
  ├── [方法C] (PMID ZZZZZZZZ) — 开源: [仓库名]
  └── ...

模型层：
  ├── [方法D] (PMID WWWWWWWW) — 开源: [仓库名]
  └── ...

训练层：
  ├── [方法E] (PMID VVVVVVVV)
  └── ...

评估层：
  ├── [方法F] (PMID UUUUUUUU)
  └── ...
```

### 4. 构建方法全景脑图

Mermaid mindmap展示所有可复用组件及其来源，按Pipeline阶段组织，叶节点标注PMID/DOI。

## 第三阶段：多路径方案生成

### 逐章生成规则

**第1章 问题定义**：
- 直接使用第一阶段输出
- 保持子问题树的完整性

**第2章 方法全景脑图**：
- 直接使用第二阶段输出
- Mermaid mindmap格式

**第3章 数据策略**：
- 严格按照 `pipeline-design.md` 数据层模板
- 每个步骤至少2个方案Fork
- Fork选择标注适用场景

**第4章 模型架构设计**：
- 严格按照 `pipeline-design.md` 模型层模板
- **必须包含Mermaid schematic**
- 3个架构变体Fork（推荐/轻量/激进）
- 每个模块标注来源(PMID/DOI) + 参考开源仓库

**第5章 训练策略**：
- 严格按照 `pipeline-design.md` 训练层模板
- 多方案对比表

**第6章 Benchmark与评估**：
- 严格按照 `pipeline-design.md` 评估层模板
- Baselines有选择理由
- 可自创评估指标
- Protocol步骤明确

**第7章 挑战与B计划**：
- 识别3类风险：方法风险/数据风险/工程风险
- 每个风险有B计划
- 失败判定标准明确
- 鼓励"降低难度也能发论文"的简化方案

**第8章 方法缝合溯源表**：
- 每个方案组件→来源论文→原文引用→开源情况→在方案中的位置
- 完整可追溯

---

## 方案生成质量控制

### 缝合检查：
- [ ] 每个缝合模块有原文方法来源(PMID/DOI)
- [ ] 每个缝合模块有原文引用
- [ ] 缝合点有具体适配方案
- [ ] 缝合策略明确标注
- [ ] 无虚构的方法组件

### 完整性检查：
- [ ] 问题精确化完成（子问题树）
- [ ] 方法全景脑图完整
- [ ] 数据层2+方案Fork
- [ ] 模型层Mermaid schematic + 3个变体Fork
- [ ] 训练层多方案对比表
- [ ] 评估层Baselines+Metrics+Protocol
- [ ] 风险层B计划
- [ ] 缝合溯源表完整

### 可实现性检查：
- [ ] 有开源仓库引用的方法标注了仓库名
- [ ] 无开源实现的方法标注了可复现性评估
- [ ] 方案的技术栈兼容性合理
- [ ] GPU/数据需求可满足
