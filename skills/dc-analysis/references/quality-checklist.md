# 反幻觉与质量检查清单

## 常见幻觉模式规避

| 模式 | 错误做法 | 正确做法 |
|-----|---------|---------|
| 创新点编造 | 添加作者未声明的创新贡献 | 仅提取原文中作者明确宣称的创新 |
| 问题夸大 | 夸大问题严重程度 | 引用原文的局限性陈述 |
| 对比伪造 | 创建不存在的对比优势 | 仅引用Discussion Step 2中的实际对比 |
| 方案编造 | 发明作者未建议的解决方案 | 仅引用Discussion Step 4中的建议 |
| 头脑风暴脱离原文 | 无原文依据的创新外推 | `[模型归纳·头脑风暴]` 必须附推理链 |
| 抑制标注 | 不标注不确定信息 | 标注 `[未明确给出]` / `[不确定]` |
| 创新贡献膨胀 | 将多个作者的贡献混为一谈 | 精确标注每个创新的来源论文 |

## 反幻觉检查

- [ ] 所有观点都有 `[原文声明]` 或 `[模型归纳]` 标签
- [ ] 所有创新声明来自原文
- [ ] 所有问题有局限性或新问题原文支撑
- [ ] 所有对比优势来自Discussion Step 2
- [ ] 所有原文引用使用原文英文 `「...」` 包裹
- [ ] 所有不确定信息标注 `[未明确给出]` 或 `[不确定]`
- [ ] 没有编造任何论文未声明的内容

## 创新提取质量检查

- [ ] 每篇论文的创新声明已提取
- [ ] 每篇论文的问题（局限性+新问题）已识别
- [ ] 问题-创新配对完整
- [ ] `[开放问题]` 正确标记（有真实问题但无创新）
- [ ] `[创新动机未明确]` 正确标记（有创新但无明确问题驱动）

## 讨论解析质量检查

- [ ] 每篇论文的Discussion解析为五步结构
- [ ] 步骤归属有位置证据
- [ ] 不确定步骤标注 `[步骤归属不确定]`
- [ ] 讨论质量评级合理

## 头脑风暴边界检查 ⭐

- [ ] 每个 `[模型归纳·头脑风暴]` 有原文线索
- [ ] 每个头脑风暴创新点有完整推理链
- [ ] 推理链包含：原文证据 → 中间推理 → 创新方向
- [ ] 信心程度标注（高/中/低）
- [ ] 可行性分析具体（非泛泛而谈）
- [ ] 可缝合方法有明确来源
- [ ] 无脱离原文的天马行空
- [ ] 无已被原文否定过的方向

## 文献可追溯性

- [ ] 每篇文献完整引用格式
- [ ] 每个观点有来源PMID或DOI标识
- [ ] 参考文献索引完整

## 可视化质量检查

- [ ] 所有图表使用Mermaid语法（无ASCII替代）
- [ ] 问题-创新逻辑链图有颜色编码和关系标注
- [ ] 创新机会脑图有原文引用
- [ ] 问题依赖网络有根因/衍生标注
- [ ] 每个图表有图例说明

## 内容完整性

- [ ] 聚焦主题深度挖掘完成
- [ ] 开放问题与创新机会章节有实质内容
- [ ] **无"文献逐篇分析"章节**
- [ ] 报告章节结构符合output-format.md
