# 创新点与问题提取规则

## 创新点提取

从三个章节语义理解作者的创新声明，不硬性关键词匹配，但以下信号短语作为参考辅助识别。

### 从Abstract提取创新声明

**关注**：作者宣告的核心贡献

**辅助信号**：
- "we propose" / "we present" / "we introduce" / "we develop"
- "novel" / "new" / "first time" / "first"
- "to address" / "to solve" / "to overcome"
- "our contribution" / "main contribution"

**提取要求**：
- `[创新声明]` 标签
- 含原文英文（「...」包裹）
- 含位置：(PMID XXXXXXXX)
- `[模型归纳]` 分析该创新在领域中的位置

**输出格式**：
```
[创新声明] 「作者宣告的核心贡献原文」(PMID XXXXXXXX)
[模型归纳] 该创新解决的现实问题、在领域中的独特位置
```

### 从Discussion提取对比优势

**关注**：作者展示的与前人工作的差异和超越

**辅助信号**：
- "compared with" / "in contrast to" / "unlike"
- "our method outperforms" / "superior to" / "better than"
- "different from" / "as opposed to"
- "overcomes the limitation of"

**提取要求**：
- `[对比优势]` 标签
- 含对比对象（prior work / method）
- 含位置：(PMID XXXXXXXX)
- `[模型归纳]` 分析该优势是否令人信服

**输出格式**：
```
[对比优势] 「与前人X方法相比的优势原文」(PMID XXXXXXXX)
对比对象：[识别出的前人工作/方法]
[模型归纳] 该对比优势的可信度评估、优势的量化/定性基础
```

### 从Conclusion提取价值升华

**关注**：作者拔高工作意义的陈述

**辅助信号**：
- "provides a new perspective" / "opens up a new avenue"
- "paves the way for" / "sheds new light on"
- "has implications for" / "contributes to"

**提取要求**：
- `[价值升华]` 标签
- 含位置：(PMID XXXXXXXX)
- `[模型归纳]` 分析该升华是否有实质支撑

**输出格式**：
```
[价值升华] 「作者拔高工作价值的原文」(PMID XXXXXXXX)
[模型归纳] 该价值升华是否有实质证据支撑、或更多是修辞性拔高
```

## 真实问题提取

### 问题来源1：Discussion Step 3（局限性）

**这是最可靠的问题来源** — 作者自己诚实地承认了不足。

**提取要求**：
- 标注为 `[原文声明]`（因为作者明确说了）
- 含局限性原文 + 位置
- 分类：方法论局限 / 数据局限 / 范围局限 / 概念局限
- `[模型归纳]` 评估该局限的严重程度和对领域的阻碍

### 问题来源2：Discussion Step 5（新问题）

**开放问题的主要来源** — 研究中提出的新方向。

**提取要求**：
- 若作者明确陈述了未解决问题 → `[原文声明]`
- 若从多处推断出隐藏问题 → `[模型归纳]`
- 标注开放性程度：完全开放 / 部分探索 / 已有初步方案
- `[模型归纳]` 分析该问题为何未被解决（技术瓶颈？数据缺失？认知盲区？）

### 问题来源3：跨文献矛盾发现

**当多篇论文的结论相互矛盾或冲突时**，这本身就是一个问题信号。

**提取要求**：
- 标注为 `[模型归纳]`（因为可能无作者明确讨论该矛盾）
- 明确列出矛盾双方的原文声明
- `[模型归纳]` 分析矛盾根源、可能的解决路径

## 问题-创新配对

### 配对规则

| 场景 | 标记 | 含义 |
|------|------|------|
| 问题P有创新I解决 | 正常配对 | 已解决问题 |
| 问题P无对应创新 | `[开放问题]` | **关键创新机会** |
| 创新I无明确问题 | `[创新动机未明确]` | 需审查该创新的真实贡献 |
| 旧问题→创新→新问题 | 逻辑链 | 创新产生的新问题 |

### 跨文献配对

同一个问题可能被多篇论文的不同创新点解决，也可能没有任何论文解决：

```
问题: "IDR的构象采样效率低"
├── 创新A: 基于深度学习的快速构象生成 (PMID XXXXXXXX, 2024)
│   └── 新问题: 生成构象的物理合理性如何保证？[开放问题]
├── 创新B: NMR约束的粗粒化模拟 (PMID YYYYYYYY, 2025)
│   └── 新问题: 粗粒化精度上限？[开放问题]
└── 未解决: 如何在保持全原子精度的同时大幅提升采样效率？[开放问题]
```

## 真实创新头脑风暴

### 核心目标

**找出原文中没做、领域中没人做、但确实该做的方向。**

### 允许条件

`[模型归纳·头脑风暴]` 必须同时满足：
1. **有原文线索**：至少一篇论文的Discussion中提到相关局限或新问题
2. **有推理链**：从原文证据到创新方向的完整推理
3. **可实现**：基于现有方法可缝合、可扩展
4. **标记信心**：高/中/低

### 输出格式

```
[模型归纳·头脑风暴] 创新方向：[方向描述]
推理链：
  1. [原文声明] 「原文中的局限/新问题陈述」(PMID XXXXXXXX)
  2. [中间推理] 从该局限可以推演出...
  3. [创新方向] 因此，可以尝试将X方法与Y方法缝合来解决Z问题
可行性分析：[为什么这个方向可实现]
信心程度：高/中/低
相关文献支撑：[哪些论文的理论/方法可被缝合]
```

### 禁止条件

- ❌ 无任何原文线索的凭空想象
- ❌ 与原文结论直接矛盾的"创新"
- ❌ 科幻级别的远未来方向（如"用通用AI自动设计所有药物"）
- ❌ 已被原文明确尝试并否定过的方向
