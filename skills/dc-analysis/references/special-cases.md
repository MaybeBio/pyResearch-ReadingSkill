# 特殊情况处理

## Abstract-only论文

**可提取**：
- 创新声明（从Abstract）

**不可提取**：
- Discussion五步结构 → `[仅有Abstract，无法解析讨论]`
- 局限性 → `[仅有Abstract，无法提取局限性]`
- 对比优势 → `[仅有Abstract，无法提取对比优势]`
- 新问题 → `[仅有Abstract，无法提取新问题]`

## Discussion-only论文（无Conclusion）

**可提取**：
- Discussion五步结构
- 局限性（Step 3）
- 对比优势（Step 2）
- 新问题（Step 5）

**不可提取**：
- 价值升华 → `[仅有讨论，无法提取结论中的价值升华]`

## Conclusion-only论文（无Discussion）

**可提取**：
- 概括内容（Part 1）
- 主要结论（Part 2）
- 价值升华（Part 3）

**不可提取**：
- Discussion五步结构 → `[仅有结论，无法解析讨论]`
- 局限性详细陈述 → `[仅有结论，局限性信息可能不完整]`
- 具体对比优势 → `[仅有结论，缺乏详细对比]`

## Discussion-Conclusion合并

- 使用五步+三段框架解析合并章节
- 标注 `[讨论-结论合并]`
- 区分讨论内容和结论内容

## 无intro-analysis输出文件

**处理方式**：
1. 从Abstract+Discussion+Conclusion自动检测主题
2. 向用户展示自动检测的主题列表
3. 请用户确认哪些主题是他们关注的
4. 标注主题来源为 `[自动检测，非intro-analysis继承]`

**局限性**：
- 自动检测的主题可能与intro-analysis不一致
- 建议用户后续提供intro-analysis输出以对齐

## 讨论质量极差

**判定**：仅有重述结论，无具体对比，局限性浅显或缺失

**处理**：
- 讨论质量评级为"较差"
- 标注可提取信息的局限性
- 该论文的问题和创新贡献权重调低

## 跨论文创新矛盾

**当两篇论文声称不同创新点解决了同一问题**：

**处理**：
- 同时记录两篇论文的创新声明
- `[模型归纳]` 分析矛盾的根源
- 标注为跨文献矛盾

**输出格式**：
```
[原文声明] paper_A声称「X方法解决了Y问题」(paper_A, Discussion Step 2)
[原文声明] paper_B声称「Z方法解决了Y问题」(paper_B, Discussion Step 2)
[模型归纳] X和Z方法对Y问题的解决方案截然不同，矛盾可能源于：1) 评估基准差异，2) Y问题的定义不同，3) 其中一方结果不可复现。需要第三方的基准对比来裁决。
```

## 同一问题无任何创新方案

**这是最有价值的发现** — 标记为 `[开放问题]`。

**处理**：
- 在报告中高亮标注
- 重点进行 `[模型归纳·头脑风暴]`
- 在创新机会脑图中放在显著位置

## 头脑风暴创新与原文冲突

**检查**：头脑风暴的缝合方案是否已被任何论文尝试并否定？

**处理**：
- 在头脑风暴前检索所有论文中是否有尝试过类似方向
- 若已被否定 → 不提出该方向，记录为 `[已探索但被放弃]`
- 若未被尝试 → 正常提出

## 文献数量过少（<3篇）

- 在报告中标注文献数量有限
- 仍然执行完整分析
- 头脑风暴时标注 `[注意：基于有限文献的推断]`
