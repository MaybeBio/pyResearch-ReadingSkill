# 双标签系统与引用格式

## 双标签系统

| 标签 | 含义 | 使用场景 |
|-----|------|---------|
| `[原文声明]` | 论文明确声明的内容 | 作者直接表述的观点、发现、局限性 |
| `[模型归纳]` | 基于原文推理的分析 | 从原文归纳但作者未明确说的分析 |
| `[创新声明]` | 作者宣称的创新贡献 | Abstract/Discussion中的创新宣告 |
| `[对比优势]` | 与前人工作的具体对比 | Discussion Step 2中的比较 |
| `[价值升华]` | 对工作意义的拔高 | Conclusion Part 3中的展望 |
| `[开放问题]` | 有真实问题但无已发表创新 | 标记为创新机会 |
| `[创新动机未明确]` | 创新无明确问题驱动 | 需审查该创新的真实贡献 |
| `[模型归纳·头脑风暴]` | 基于原文外推的创新方向 | 必须附推理链和信心程度 |
| `[步骤归属不确定]` | Discussion步骤划分不确定 | 段落边界模糊时使用 |

## 不确定性标记

| 标记 | 含义 |
|------|------|
| `[未明确给出]` | 论文中未提及的字段 |
| `[不确定]` | 信息有歧义 |
| `[讨论-结论合并]` | Discussion和Conclusion在原文中是同一章节 |
| `[仅有讨论]` | 论文有Discussion但无独立Conclusion |
| `[仅有结论]` | 论文有Conclusion但无独立Discussion |

## 原文-分析配对原则 ⭐

### 核心规则

**原文引用是证据，分析归纳是价值。二者缺一不可。**

> 每条 `[原文声明]` 必须搭配 `[模型归纳]` 分析。禁止连续堆砌原文引用而不作分析。

### 正确模式

```
[原文声明] 作者指出「原文内容」（paper_id）
    ↓
[模型归纳] 基于上述原文的推理分析，说明推理依据
```

### 禁止模式

```
❌ 错误：原文堆砌
[原文声明] 作者A指出「...」
[原文声明] 作者B指出「...」
[原文声明] 作者C指出「...」
（连续3条原文引用无分析 — 这不是综述，是摘抄）
```

### 配对规则

| 规则 | 说明 |
|-----|------|
| **1配1** | 每条`[原文声明]`必须有紧随的`[模型归纳]` |
| **先原文后分析** | `[原文声明]` → `[模型归纳]`，不可逆序 |
| **分析有据** | `[模型归纳]`必须说明从原文哪部分推演出该结论 |
| **禁止连续堆砌** | 最多连续2条`[原文声明]`后必须插入`[模型归纳]` |
| **同主题多文献** | 可将多篇相关原文并列引用，后跟综合`[模型归纳]` |

## 引用格式

### 行内引用（短句）
```
[原文声明] 作者指出「Deep learning has demonstrated remarkable performance」(PMID 12345678)
```

### 引用块（长段落）
```
> "Deep learning has demonstrated remarkable performance in medical image analysis"
> (PMID 12345678)
```

### 引用三要素
1. **来源文献**：完整引用格式（PMID only）
2. **原文内容**：保留英文，用「...」包裹
3. **文献标识**：DOI或PMID作为唯一标识

### 格式

| 位置 | 格式 |
|-----|------|
| Discussion Step X | `(PMID 12345678)` |
| Conclusion Part Y | `(PMID 12345678)` |
| Abstract | `(PMID 12345678)` |

## 头脑风暴引用格式

### 必须包含

```
[模型归纳·头脑风暴] 创新方向：[方向描述]
推理链：
  1. [原文声明] 「原文中的局限/新问题」(PMID XXXXXXXX)
  2. [中间推理] 从该局限推演出...
  3. [创新方向] 具体可操作的创新方案
可行性分析：[为什么可实现]
可缝合方法：[方法1(PMID XXXXXXXX) + 方法2(PMID YYYYYYYY)]
信心程度：高/中/低
```

### 信心程度标准
- **高**：原文明确暗示该方向，且已有类似缝合案例
- **中**：原文暗示了需求，缝合路径合理但需验证
- **低**：方向合理但实现路径有显著不确定性

## 分析深度层级

### Level 1: 直接归纳
对单条原文的直接归纳。

### Level 2: 跨文献关联
将多条原文关联、对比。

**示例**：
```
[原文声明] PMID A指出「MD模拟的瓶颈在于采样效率」(PMID A)
[原文声明] PMID B指出「生成式ML可大幅提升构象生成效率」(PMID B)
[模型归纳] PMID A的问题(采样效率低)可被PMID B的方法(生成式ML)部分解决，二者存在缝合机会。但PMID B未讨论生成构象的物理合理性——这是缝合时需要解决的物理约束问题。
```

### Level 3: 深层推理与创新外推
基于原文进行跨文献综合推理，形成新的创新理解。

**示例**：
```
[原文声明] PMID C指出「pLM捕捉了序列中的进化信息但缺乏物理可解释性」(PMID C)
[原文声明] PMID D指出「基于物理的MD力场可解释性强但受限于采样」(PMID D)
[模型归纳·头脑风暴] 创新方向：物理约束的pLM生成式构象采样
推理链：
  1. pLM可编码构象偏好(PMID C的发现) → 可作为构象生成的先验分布
  2. MD力场能量(PMID D的方法) → 可作为生成后的物理合理性过滤器
  3. 缝合方案：pLM生成候选构象 → MD能量函数打分筛选 → 保留物理合理的构象集合
可缝合方法：ProtT5 embedding(PMID 34567890) + AMBER能量函数(PMID 45678901)
信心程度：中
```
