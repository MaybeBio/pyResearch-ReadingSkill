# 输出格式与保存规则

## 输出文件结构

### 完整报告模板

```markdown
# [领域名称] 创新点分析报告

## 执行摘要 (Executive Summary)
- 分析文献数量：N篇
- 识别真实问题：X个（开放问题：Y个）
- 提取创新声明：Z个
- 聚焦主题：[用户选择的主题名称]
- 核心开放问题：[列出2-3个最重要的]
- 数据来源：Abstract + Discussion + Conclusion
- 主题先验来源：[intro-analysis输出文件路径]

## 0. 创新点全景 (Innovation Overview)

### 创新机会脑图
[Mermaid mindmap — 按开放问题/可缝合方向/未尝试方向全景展开]

### 统计概览
| 类别 | 数量 | 说明 |
|------|------|------|
| 提取论文数 | N | 含Discussion+Conclusion的论文 |
| 创新声明 | Z | 从Abstract/Discussion/Conclusion提取 |
| 已解决问题 | A | 有创新方案对应 |
| 开放问题 | B | 无已发表创新方案 — 创新机会 |
| 头脑风暴方向 | C | 基于原文外推的创新机会 |

## 1. 主题深度挖掘 (Theme Deep Dive)

### 1.X [用户选择的主题名称]

#### 主题的真实问题汇总
- 每个问题：[原文声明]（原文+位置）+ [模型归纳]（问题分析）
- 问题来源标注：Step 3 局限性 / Step 5 新问题 / 跨文献矛盾
- 问题类型标注：方法论 / 数据 / 范围 / 概念
- 严重程度：高 / 中 / 低

#### 主题的创新点汇总
- 每个创新：[创新声明] + [对比优势] + [价值升华]
- 该创新解决了什么问题
- 该创新产生了什么新问题

#### 主题的问题-创新配对
- 问题P1 → 创新I1 → 新问题P1_new（或标记为最终解决）
- 无创新对应的问题 → [开放问题]

## 2. 问题-创新逻辑链 (Problem-Innovation Logic Chain) ⭐

**[核心可视化]**：旧问题→已有创新→产生的新问题→可能的创新方向

### 2.1 逻辑链总图
[Mermaid graph — 全主题问题-创新逻辑链，颜色编码]

### 2.2 逐问题展开
每个重要问题独立展示逻辑链：
- 问题描述 [原文声明] + 位置
- 已有解决方案（如有）
- 方案产生的新问题（如有）
- 创新机会 [模型归纳·头脑风暴] + 推理链 + 信心

## 3. 问题依赖网络 (Problem Dependency Network)

### 3.1 问题依赖图
[Mermaid graph — 问题之间的根因/层级/相关关系]

### 3.2 问题依赖表
| 问题ID | 描述 | 类型 | 根源问题 | 依赖问题 | 严重程度 |
|-------|------|------|---------|---------|---------|

## 4. 开放问题与创新机会 (Open Problems & Innovation Opportunities)

### 4.1 开放问题清单
| 问题ID | 问题描述 | 来源论文 | 为何未被解决 | 涉及主题 |
|-------|---------|---------|------------|---------|

### 4.2 真实创新头脑风暴 ⭐
按开放问题逐一展开：
```
[模型归纳·头脑风暴] 创新方向：[方向描述]
针对开放问题：[问题ID]
推理链：
  1. [原文声明] 「原文中的局限/新问题陈述」PMID XXXXXXXX
  2. [中间推理] 从该局限可以推演出...
  3. [创新方向] 将X方法与Y方法缝合来解决Z问题
可缝合的方法来源：[列出可复用的已有方法]
可行性分析：[为什么这个方向可实现]
信心程度：高/中/低
```

### 4.3 缝合路线图
[Mermaid graph — 展示哪些已有方法可以缝合到哪些开放问题上]

## 5. 参考文献索引 (Reference Index)

### 5.1 文献元信息表
| ID | PMID | 标题 | 作者 | 年份 | DOI | 讨论质量 |
|----|------|------|------|------|-----|---------|
| paper_1 | ... | ... | ... | ... | ... | 优秀/一般/较差 |

### 5.2 引用格式说明
- 完整格式：作者, 年份. 标题. 期刊. DOI/PMID
- 信息缺失处理：使用 `[未明确给出]`

## 6. 可视化图例 (Visualization Legend)
- 红色节点 = 现有真实问题
- 绿色节点 = 已有创新方案
- 黄色节点 = 创新产生的新问题
- 蓝色虚线节点 = 头脑风暴的创新机会
- 实线箭头 = 已验证的因果关系
- 虚线箭头 = 推演的创新路径

## 7. 反质量检查 (Quality Checklist)
- [ ] 所有观点都有 `[原文声明]` 或 `[模型归纳]` 标签
- [ ] 所有原文引用使用原文英文 `「...」` 包裹
- [ ] 所有不确定信息标注 `[未明确给出]` 或 `[不确定]`
- [ ] 每个问题都有来源标注（Discussion）
- [ ] 每个头脑风暴创新点都有推理链
- [ ] 无脱离原文的天马行空
- [ ] 所有图表使用Mermaid语法
- [ ] 问题-创新逻辑链图有颜色编码
- [ ] **无"文献逐篇分析"章节**（dc-analysis不包含此章节）
```

## 输出方式

### 核心原则

**只保存文件，不在对话中显示完整内容**

### 执行步骤

1. 生成完整报告
2. 直接保存到本地文件
3. 对话中仅显示简要摘要

### 保存位置

**文件命名格式**：
```
[输入文件名]_dc_[YYYYMMDD].md
```

**示例**：
- 输入文件：`IDR_dc_papers.md`
- 保存文件：`IDR_dc_papers_dc_20260523.md`

**日期格式**：YYYYMMDD

---
