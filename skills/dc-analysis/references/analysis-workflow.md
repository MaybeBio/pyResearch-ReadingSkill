# 分析流程详细说明

## 三阶段分析流程

---

## 第一阶段：逐篇DC深度解析

### 0. 文献元信息提取

- 完整引用格式生成
- 元信息验证
- 缺失信息标记 `[未明确给出]`

### 1. Discussion五步结构解析

对每篇论文的Discussion，将内容分配到对应步骤：

- **Step 1: 重述结论** — 作者最强调的发现
- **Step 2: 对比前人** — 与已有文献的异同、优势
- **Step 3: 局限性** — **真实问题的主要来源**
- **Step 4: 解决方案** — 作者建议的改进
- **Step 5: 新问题** — **开放问题的核心来源**

每步标注：`[原文声明]`（原文+位置）+ `[模型归纳]`（该步骤内容的分析意义）

不确定的步骤归属标注为 `[步骤归属不确定]`

讨论质量评级：优秀 / 一般 / 较差

### 2. Conclusion三段结构解析

- **Part 1: 概括研究内容** — 方法+问题
- **Part 2: 主要结论** — 结果+含义
- **Part 3: 展望** — 价值拔高+未来方向

每部分标注位置和标签

### 3. 创新声明提取

从Abstract、Discussion、Conclusion中语义识别：
- `[创新声明]` — 核心贡献宣告
- `[对比优势]` — 与前人对比
- `[价值升华]` — 意义拔高

不硬性关键词匹配，语义理解为主

### 4. 真实问题识别 ⭐

**三个来源**：

| 来源 | 问题性质 | 提取方式 |
|------|---------|---------|
| Discussion Step 3 局限性 | 作者承认的不足 | `[原文声明]` 直接提取 |
| Discussion Step 5 新问题 | 揭露的新方向 | `[原文声明]` 或 `[模型归纳]` |
| 跨文献矛盾 | 不同论文的矛盾发现 | `[模型归纳]` 综合分析 |

每个问题标注：类型（方法论/数据/范围/概念）、严重程度（高/中/低）

### 5. 局限性提取与分类

- 局限性原文 + 位置
- 分类：方法论 / 数据 / 范围 / 概念
- 评估对核心结论的影响
- 是否有解决方案（Step 4）
- 无解决方案的局限性 → `[开放问题]` 候选

### 6. 未来方向提取

- 作者建议的未来方向（Discussion Step 5 + Conclusion Part 3）
- 新问题（Discussion Step 5）
- 标注 `[原文声明]` + 位置

### 7. 问题-创新配对（每篇论文）

对每篇论文：
```
问题P1 → 创新I1 → 新问题P1_new
问题P2 → 无创新 → [开放问题]
创新I3 → 无明确问题 → [创新动机未明确]
```

### 8. 原文-分析配对检查

- 每条 `[原文声明]` 有配套 `[模型归纳]`
- 无连续2条以上纯原文引用
- 关键发现至少达到Level 2（跨文献关联）

## 第二阶段：跨文献综合与主题挖掘

### 1. 主题对齐

**从intro-analysis输出文件读取主题列表，向用户展示：**

```
📋 从intro-analysis继承的8个主题：
1. IDR数据库与资源建设
2. 无序性预测方法演进
3. IDR介导的蛋白-蛋白互作预测
...
```

**询问用户聚焦哪个主题深入挖掘**（单选或多选均可）

### 2. 全域文献深度挖掘 ⭐

**intro-analysis的主题是组织框架，不是文献边界。聚焦主题X时，全局文献均可调用。**

对选定主题：
- **不从主题标签筛选论文**：调用全部输入文献（含Abstract-only），只要讨论与该主题相关即可引用
- **跨主题围攻**：从相邻主题的论文中提取与该主题相关的讨论，形成多维交叉分析
- 汇总该主题的真实问题（来源可跨越所有主题标签的论文）
- 汇总该主题的创新声明
- 建立问题-创新配对关系
- **问题多态细分**：每个大问题拆解为多个子问题，沿细分赛道逐层深入

### 3. 开放问题识别

梳理该主题内：
- 哪些问题有已发表的创新方案？
- 哪些问题没有任何创新方案？→ `[开放问题]`（核心创新机会）
- 哪些创新指出了新的未解决问题？
- 从相邻主题的开放问题中找到与本主题的交叉地带→ 高价值创新机会

### 4. 跨文献+跨主题创新对比

同一问题被不同论文（可能分布在不同主题标签下）解决了吗？
- 相同问题、不同创新 → 对比创新优劣
- 相同问题、无创新 → 开放问题
- 不同主题共享同一问题 → 交叉创新机会（知识浓度渗透）

### 5. 问题依赖网络构建

分析问题之间的根因关系（跨主题维度）：
- 根因问题 → 衍生问题（因果关系）
- 平行问题（相关但不因果）
- 矛盾问题（互相冲突）
- 跨主题问题链：主题A的问题 → 诱发了主题B的问题

### 6. 真实创新头脑风暴 ⭐

**这是本skill的核心价值输出阶段。**

对每个 `[开放问题]`：
1. 列出原文中对这个问题的描述（来源论文+位置，不限主题标签）
2. 分析为什么这个问题未被解决
3. 识别可缝合的已有方法（从全域文献中找，跨主题缝合）
4. **提出具体的、可实现的创新方向**

**头脑风暴检查**：
- [ ] 有原文线索支撑
- [ ] 推理链完整清晰
- [ ] 基于可实现的方法缝合
- [ ] 跨主题文献已被检索（非仅限单一主题标签的论文）
- [ ] 不是已被尝试并否定的方向
- [ ] 不是脱离原文的天马行空

## 第三阶段：生成创新分析报告

### 输出文件命名

**格式**：`[输入文件名]_dc_[YYYYMMDD].md`

### 保存规则

- 完整报告直接保存到文件
- 对话中仅显示简要摘要
- 不包含"文献逐篇分析"章节

### 对话摘要格式

```
✅ 分析完成

📊 分析统计
- 文献数量：N篇
- 识别真实问题：X个（开放问题：Y个）
- 提取创新声明：Z个
- 聚焦主题：[主题名称]

📁 输出文件
- 路径：[文件路径]
- 格式：Markdown (.md)

💡 核心发现
- 最重要的开放问题
- 最有前景的创新方向
- 可缝合的方法组合
```

## 分析流程质量控制

### 第一阶段检查：
- [ ] 每篇论文的Discussion被正确解析为五步结构
- [ ] 每篇论文的创新声明被提取
- [ ] 每篇论文的问题（局限性+新问题）被识别
- [ ] 问题-创新配对完成
- [ ] 每条 `[原文声明]` 有配套 `[模型归纳]`

### 第二阶段检查：
- [ ] 主题对齐正确
- [ ] 聚焦主题深度挖掘完成
- [ ] 开放问题清晰识别
- [ ] 问题依赖网络合理
- [ ] 头脑风暴创新点有原文线索+推理链

### 第三阶段检查：
- [ ] 文件命名正确
- [ ] 报告结构完整
- [ ] 不包含"文献逐篇分析"章节
- [ ] 简要摘要简洁
