# 特殊情况处理

## Abstract-only 论文处理

### 可提取内容

从Abstract可提取：
- 研究问题 `[原文声明]`
- 方法 `[原文声明]`
- 主要贡献 `[原文声明]`
- 简要结果 `[原文声明]`

### 不可提取内容

以下内容标注为 `[仅有Abstract，无法提取]`：
- 历史脉络 → `[仅有Abstract，无法提取历史脉络]`
- 研究现状分类 → `[仅有Abstract，无法提取研究现状]`
- 研究缺口 → `[仅有Abstract，无法提取研究缺口]`
- 关键参考文献 → `[仅有Abstract，无法提取参考文献]`
- 详细论证 → `[仅有Abstract，无法提取详细论证]`
- 引言结构分析 → `[仅有Abstract，无法分析引言结构]`

### 处理方式

1. **只提取Abstract中明确陈述的内容**，标注为 `[原文声明]`
2. **标记为部分分析**，在文献分析中明确说明
3. **不参与**需要历史脉络或研究缺口的综合分析
4. **只在"核心观点"部分**收录其提出的方法或贡献

### 输出示例

```markdown
### 6.2 [完整引用格式] - 有限分析（仅有Abstract）

**数据来源**：`[仅有Abstract，缺失Introduction]`

**可提取内容**：
- 研究定位 `[原文声明]` + 证据
  - 「This paper proposes a novel approach for...」 (paper_2, Abstract)
- 核心观点 `[原文声明]` + 证据
  - 「Our method achieves state-of-the-art performance」 (paper_2, Abstract)
- 主要贡献 `[原文声明]` + 证据
  - 「We contribute: 1) ... 2) ... 3) ...」 (paper_2, Abstract)

**无法提取**：
- 历史脉络 `[仅有Abstract，无法提取历史脉络]`
- 研究现状 `[仅有Abstract，无法提取研究现状]`
- 研究缺口 `[仅有Abstract，无法提取研究缺口]`
- 关键参考文献 `[仅有Abstract，无法提取参考文献]`
- 引言结构分析 `[仅有Abstract，无法分析引言结构]`
```

## 未提供完整元数据

### 问题

论文ID缺失或元信息不完整

### 处理方式

1. **如果论文ID缺失**：使用 `paper_1`, `paper_2` 等临时编号
2. **如果部分信息缺失**：使用 `[未明确给出]` 标记
3. **自动生成引用**：使用可用信息生成引用格式

### 示例

```markdown
---
id: paper_1
title: 论文标题
authors: 作者列表
year: 年份
journal: [未明确给出]
doi: [未明确给出]
pmid: [未明确给出]
---
```

## 引言质量差异

### 引言规范

- 按标准流程提取所有信息
- 完整执行四阶段分析

### 引言简单

- 只提取基本信息，不做过度推断
- 标注引言质量为 ⚠️一般 或 ❌较差
- 在第二阶段综合分析中标注限制

## 跨语言论文

### 处理方式

1. **标注语言**：在文献元信息中标注原文语言
2. **提取可理解的部分**：提取原文中可理解的内容
3. **保留原文表述**：关键观点保留原文语言
4. **标注不确定性**：对理解有困难的内容标注为 `[不确定]`

---

## 重复引用

### 问题

同一观点在不同论文中反复引用

### 处理方式

1. **在观点论战分析中**统一归类
2. **列出所有支持文献**：paper_a, paper_b, paper_c
3. **区分引用关系**：
   - 直接引用原始工作
   - 二手引用他人观点

## 循环依赖

### 问题

观点A依赖B，B又依赖C，C又依赖A

### 处理方式

1. **在观点依赖图中**标注为循环依赖
2. **使用虚线边框**：`A -.循环.-> B`
3. **添加说明文字**：`[循环依赖]`

### 示例

```mermaid
graph TD
    A[「观点A」<br/>(paper_1)] -.循环.-> B[「观点B」<br/>(paper_2)]
    B -.循环.-> C[「观点C」<br/>(paper_3)]
    C -.循环.-> A

    style A stroke:#ff0000,stroke-width:2px,stroke-dasharray: 5 5
    style B stroke:#ff0000,stroke-width:2px,stroke-dasharray: 5 5
    style C stroke:#ff0000,stroke-width:2px,stroke-dasharray: 5 5
```

## 二手引用信息不全

### 问题

论文引用的原始工作在引言中描述不详

### 处理方式

1. **标注为 `[引用文献未详述]`**
2. **保留有限信息**：只保留引言中明确提到的内容
3. **不编造**：绝不猜测或补充信息

### 示例

```markdown
**关键参考文献**：
- [Smith et al., 2015] (cited_by: paper_1, type: 引用)
  - 「Smith et al. proposed a novel approach...」 [引用文献未详述]
  - (paper_1, Introduction第3段)
```

## 时间线不一致

### 问题

同一事件在不同文献中有不同年份表述

### 处理方式

1. **同时记录所有表述**
2. **标注差异**
3. **使用表格对比**

### 示例

| 事件 | 文献A表述 | 文献B表述 | 差异 |
|-----|---------|---------|------|
| 方法提出 | 2015年 | 2016年 | 年份差异1年 |

## 观点表述模糊

### 问题

观点表述不明确或有歧义

### 处理方式

1. **标注为 `[不确定]`**
2. **说明歧义原因**
3. **不过度推断**

### 示例

```markdown
**核心观点**：
- 「The method shows promising results」 [不确定 - 未提供具体指标]
  - (paper_1, Introduction第5段)
```

## 文献数量不足

### 问题

文献数量少于3篇，难以进行综合分析

### 处理方式

1. **标注局限性**：在执行摘要中说明
2. **仍执行完整分析**：按标准流程处理
3. **结果仅供参考**：在结论中说明样本量限制

### 示例

```markdown
## 执行摘要
- 分析文献数量：2篇 [样本量有限]
- 分析局限性：文献数量较少，结论仅供参考
```

---

## 文献时间跨度过大

### 问题

文献时间跨度超过10年，研究范式可能已发生重大变化

### 处理方式

1. **标注时间跨度**
2. **分析范式转换**：使用state diagram展示
3. **分层分析**：按时期分类分析

### 示例

```markdown
## 执行摘要
- 时间跨度：1995 - 2025（30年）
- 研究范式转换：传统方法 → 深度学习 → 大模型时代
```

## 输入文件格式错误

### 问题

输入文件不符合预期格式

### 处理方式

1. **尝试修复**：尽可能解析可用内容
2. **标注错误**：明确指出格式问题
3. **继续分析**：使用可用的部分

### 示例

```markdown
⚠️ 输入文件格式警告：
- paper_1：元信息不完整，部分字段缺失
- paper_2：缺少Introduction，仅有Abstract
- 继续使用可用内容进行分析...
```
